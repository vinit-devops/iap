### Added

- `@iis/migrate` 0.1.0 — migration importers (roadmap Phase 18, M18.1–M18.4 — **completes Phase 18, and the entire roadmap**): `importKubernetes` translates Kubernetes manifests into IIS **through the operation gate** — Deployment/StatefulSet/ReplicaSet → Service, Job/CronJob → Job, PersistentVolumeClaim → Volume, Ingress → Gateway, Secret → Secret — so the imported result is validated IIS; constructs it cannot faithfully map (ConfigMap, a K8s Service) are reported as `unmapped` with a reason, **never guessed** (precision over recall). Deterministic. Terraform/CloudFormation/Pulumi/Crossplane/live-resource importers implement the same `ImportResult` contract. CI/CD integrations (GitHub Actions/GitLab/Jenkins/Argo/Backstage shelling out to the CLI; the control plane's `prChecks` as the hosted git-app form), signed digest-pinned registries + certification (over the Phase-6 package/trust model), and the open-governance model (implementation-independent spec/conformance, IEP process, TSC, LTS) are documented in `docs/guides/ecosystem.md`
- `@iis/ai-review` 0.1.0 — the AI review engine and guardrails per spec ch. 19 (roadmap Phase 17, M17.1–M17.2 — **completes Phase 17**): the "AI" is a deterministic review LENS, never a source of nondeterministic or binding output — the platform operates fully without it. `reviewDocument(model)` folds the security + compliance engines and deterministic best-practice rules (`max-availability-without-backup`, `orphaned-data-resource`) into **explainable, source-cited, advisory** `ReviewItem`s (rule id + explanation + suggestion + citations). `assertGuardrails` fails closed on the two hard rules — every item is **advisory (never blocking/deny**, §20.2.3) and **cites at least one source** — so an AI review can never become a blocking gate or an unsourced claim. The engine never mutates; acceptance rides the intent-compiler `recommend → acceptRecommendations → gate` (TB-3 citation-gated), and a model change cannot alter an approved plan (envelopes bind `planId`, Phase 7). Org context threads MCP enterprise snapshot ids as citations; the deterministic review is itself the evaluation reference
- `@iis/control-plane` 0.1.0 — the enterprise control-plane core (roadmap Phase 16, M16.1–M16.3 — **completes Phase 16**): multi-tenant project isolation, RBAC (`viewer`/`author`/`approver`/`admin` × `view`/`author`/`approve`/`deploy`/`administer`), a **separation-of-duties approval engine** (`approve` — the author of a change can never approve it; only an `approve`-holding role may; every approval is recorded evidence tying a deployment to identity), an append-only audit log (`ControlPlane.authorize` records every allow/deny), org-distributed policy-pack inheritance per project, and **PR checks** (`prChecks(base, head)`) reporting intent / cost (monthly delta + budget breaches) / security (risk change + new findings) / compliance (new control violations) deltas — each pass/fail with an overall gate — so a git application blocks a PR that regresses posture. Reuses the reference cost/security/compliance engines; the HTTP API, git application, and central registries are thin clients over the core (the state service is `@iis/state`)
- `@iis/designer` 0.1.0 — the headless visual-designer session (roadmap Phase 15, M15.1–M15.3 — **completes Phase 15**): the testable core every visual surface drives, enforcing the phase's key invariant — **the UI never becomes a second source of truth**. The canvas is a VIEW; the IIS document is the single source of truth. `DesignerSession` translates every canvas edit — `addResource`/`connect`/`setProperty`/`remove` — into a `visual-designer`-channel compiler operation and commits it through the same gate (`apply`) the CLI uses; a rejected edit leaves the document unchanged. `inspect(id)` is the property inspector (effective spec + per-field provenance); `yaml()` serializes. The produced IIS is valid and byte-deterministic; plan approval reuses `iis plan` and git diff is the document diff (no proprietary project format). The browser shell, canvas rendering, and approval/diff UX are thin clients over this session
- `@iis/state` + `@iis/deploy` 0.1.0 — the deployment, state, verification and drift engines per spec ch. 13/ch. 14 + IEP-0010 (roadmap Phase 14, M14.1–M14.4 — **completes Phase 14**). `@iis/state`: a pluggable `StateBackend` (read / CAS write / appendHistory / lock lifecycle / capabilities) with `LocalStateBackend` — **lease-based locking that fails closed on a live lease** (`LockHeldError`; concurrent state mutation prevented), CAS writes on a monotonic revision with an integrity hash (`RevisionConflictError`/`InvalidLockError`), and append-only history with the roadmap §4.4 fields (approvals, applied/failed, findings, rollback/verification). Clock injected. `@iis/deploy`: `deploy` drives an approved plan through a provider `DeploymentExecutor` with the full safety envelope — **destructive-approval gate** (§19.6), fail-closed locking, an atomic CAS state commit with **partial-state recovery** (applied objects committed, failed ids retain prior state, §14.7), post-deployment verification, `detectDrift` on the IEP-0010 two-axis taxonomy (disposition × severity), and a `rollback` framework (§14.6). Execution contains **no AI and no MCP**; a `fixtureExecutor` (with `failOn`/`driftOn`) plus the Phase-6 mock substrate's injectable failures form the failure-injection suite. The live AWS execution pilot and remote/enterprise state backends implement the same interfaces and are documented follow-ons (the deterministic verify harness excludes cloud/network)
- `@iis/mcp-server` 0.1.0 — the IIS MCP server (roadmap Phase 13, M13.2 — **completes Phase 13**): the tool surface an AI assistant or IDE drives to author and review IIS. A protocol-neutral `IisMcpServer` dispatcher over the read-only `IIS_TOOLS` registry — `iis_author` (natural-language authoring through the intent-compiler gate; returns outcome, clarifications, semantic preview, committed document, and **per-field provenance**), `iis_validate`, `iis_cost`, `iis_security`, `iis_compliance` (the same reference engines the CLI uses). Its defining property is the ch. 19 boundary: **no deployment, mutation, or provider-API tool exists** — an assistant cannot deploy or reach a provider; `assertReadOnly` fails closed at construction if a tool names a mutation verb or a non-read-only kind is introduced. VS Code (M13.1) and Cursor/Claude Code/Windsurf/JetBrains (M13.3) integrations are thin clients over this server and the Phase-4 language server, delivered as `docs/guides/ide-integration.md`
- `@iis/mcp` 0.1.0 — MCP and authoritative-knowledge framework per spec ch. 20 + IEP-0013 (roadmap Phase 12, M12.1+M12.2 — **completes Phase 12**): **knowledge in, nothing out** — MCP-backed sources enrich authoring/validation and never participate in planning or execution. `KnowledgeSource` (read-only; `null` = unavailable) with network-free `fixtureSource`; closed `KNOWLEDGE_CATEGORIES` (provider-documentation, pricing, best-practice, security-advisory, enterprise) + `TRUST_TIERS`; `SourceRegistry` (dedup, by-category); and `KnowledgeClient` that retrieves as **pinned** snapshots — reproducible without a live call, live retrieval only on explicit `refresh` (the §20.1 refresh point), unavailable sources reported never thrown (graceful degradation, §20.3). `KnowledgeSnapshot` is a content-addressed citation model (source, retrieval time, version, query, excerpt, confidence, expiry, accepted); `createSnapshot`/`citation`/`isStale` are pure with an injected clock. `groundRecommendation` builds an intent-compiler `origin: 'mcp'` recommendation grounded in snapshot ids — accepted only through the existing IEP-0013 **TB-3** citation gate, becoming explicit intent through the operation gate before planning so no uncited value reaches a deterministic plan (§20.3 plan invariance)
- `@iis/compliance` 0.1.0 — compliance engine per spec ch. 17 (roadmap Phase 11, M11.2 — **completes Phase 11**): compliance is **embedded, evaluated at the intent level**, never a claim of certification. `FRAMEWORK_BUNDLES` is the v1 registry of six versioned bundles (soc2, pci-dss-4.0, hipaa, iso27001-2022, nist-800-53-r5, cis-8.0), each a set of controls that map a control id to a deterministic rule — a field rule (dot-path leaf) or a derived graph check (`every-data-edge-has-access`, `no-admin-to-data`, `workloads-authenticated`, `no-undeclared-reachability`) — plus remediation and technical/external evidence. `evaluateCompliance(model)` gives every control a **satisfied/violated/not-applicable** disposition with contributing paths (§17.6), emits **IIS701** (control violation: framework/control/bundle-version/path) and **IIS702** (a scoped framework with nothing in scope), applies label scoping (§17.7), and carries a normative disclaimer distinguishing configuration coverage from formal certification. New CLI command **`iis compliance`** (`--file`/`--profile`/`-o human|json`) flips the last Phase-11 stub — per-control dispositions + remediation + evidence report, exit 1 on any error finding
- `@iis/security` 0.1.0 — security analysis engine per spec ch. 15 (roadmap Phase 11, M11.1): **security is derived from the canonical document, never annotated**. `deriveGrants` builds the least-privilege grant table from edge `access` attributes alone (no edge → no grant; attached to each workload's `authenticatedBy` or implicit identity, §15.3); `deriveReachability` builds the zero-trust reachability graph from `exposure` + connectivity edges (§15.4); `deriveEncryption` reports posture (§15.6). Findings `securityFindings` emit **IIS601** (public exposure on a data kind — error when a `storesDataIn` target, warning otherwise), **IIS602** (plaintext secret material in `configuration`/`x-*` — well-known token shapes + secret-named keys, §15.5), and **IIS603** (encryption downgrade under an active `pci-dss-4.0`/`soc2` framework, §15.6); IIS604 stays plan-time. `scoreRisk` gives a deterministic `none`…`critical` level and `securityReport` assembles the §15.9 review surface (`reportVersion: 1`). New CLI command **`iis security`** (`--file`/`--profile`/`-o human|json`) flips the Phase-11 stub — grant table + reachability + findings, exit 1 on any error-severity finding. `compliance` remains gated (M11.2)
