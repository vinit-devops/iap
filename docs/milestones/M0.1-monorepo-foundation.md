**Phase:** 0 — Foundation and Repository Baseline
**Milestone:** M0.1
**Status:** Completed

## Implemented

- Restructured the repository into the roadmap §8 monorepo layout. All IaP v1 spec artifacts moved under `spec/`:
  - `spec/*.md` → `spec/chapters/` (24 chapters)
  - `schema/` → `spec/schema/`
  - `examples/` → `spec/examples/`
  - `examples/mappings/` → `spec/mappings/`
  - `conformance/` → `spec/conformance/`
  - New empty roots: `packages/`, `providers/`, `extensions/`, `apps/`, `tests/{conformance,integration,determinism,security,end-to-end}`, `tools/{schema-generation,release,migration}`, `docs/{milestones,reports,adr,architecture}`, `spec/ieps/`
- Fixed every cross-reference affected by the move (chapter → README links, README → chapter/schema/example/conformance links, conformance README → chapter links and run-from-root command paths). Mechanical link check reports zero broken relative links.
- Workspace tooling:
  - `package.json` (root, private) with `build` / `test` / `test:spec` / `lint` / `format:check` / `typecheck` / `verify` scripts; pinned `packageManager: pnpm@11.5.0`, Node `>=22`
  - `pnpm-workspace.yaml` covering `packages/*`, `providers/*`, `extensions/*`, `apps/*`
  - `tsconfig.base.json` (strict TypeScript, NodeNext, declarations)
  - ESLint 9 flat config + typescript-eslint; Prettier with spec prose excluded from formatting
  - Vitest as the unit-test runner
  - `.editorconfig`, `.gitignore`, `.prettierignore`
- CI workflow at `.github/workflows/ci.yml`: install → build → lint → format check → unit tests → spec validation (`pnpm run test:spec`). Inert until the repo is pushed to a git host, by design.
- `pnpm install` verified locally: toolchain binaries resolve (prettier 3.9.4, eslint 9.39.4, tsc 5.9.3, vitest 3.2.7).

## Conformance status

Unchanged — no normative content touched. Manual ajv validation of schemas/examples was green before the move; the automated harness that re-proves it lands in M0.6.

## Architecture decisions

- pnpm workspace + plain `tsc` builds; **turbo.json deferred** (roadmap lists it as recommended; with two initial packages it adds no value — introduce when package count makes caching worthwhile). Will be recorded in ADR-0004 (monorepo-first development) in M0.2.
- Vitest over node:test/jest: TS-native, fast, single config at root.
- Spec prose (`spec/chapters`, examples, cases) excluded from Prettier so the normative text never churns from formatter upgrades.

## Specification gaps

None discovered in this milestone (gap analysis is M0.3).

## Security findings

None. No dependencies with install scripts other than esbuild (explicitly allowlisted via `allowBuilds`).

## Known limitations

- Repository is intentionally **not under git** (user directive). CI workflow, `.gitignore`, and the roadmap's PR-based workflow are staged but inactive; reviewable milestone documents replace PRs until git is enabled.
- `LICENSE` not created — license selection is a user decision (flagged in M0.2).
