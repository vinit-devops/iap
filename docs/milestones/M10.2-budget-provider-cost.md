## Design decisions taken

1. **Budget lives in `@iap/cost`, emits IAP505 (not IAP501).** Â§16.7 makes budgets
   plan-time and gives them their own code; routing through the generic policy engine would
   mislabel them. The budget evaluator reuses ch. 7 target semantics (kinds + selector).
2. **`unknown` cost is unevaluable, never a pass.** A budget over a resource (or roll-up)
   the model cannot price yields a warning, honoring Â§16.7's "never silently passed."
3. **The CLI prices with the reference model by default.** Provider-distributed cost models
   ship with mappings; wiring `iap cost --mapping` to a package's cost model is a small
   follow-on. The mock cost model is exported and tested to prove the distribution pattern.
4. **Cost stays a projection.** Nothing is written back to the document; `annotateModel`
   produces a derived copy for downstream annotation consumers (LSP hovers, plan output).
