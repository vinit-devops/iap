## Implemented

**`@iap/policy` 0.1.0** (`packages/policy/`, depends on `@iap/model` only):

### M9.1 — `evaluatePolicies(model, options?)` (spec ch. 7; validation phase 5, IAP5xx)

- **Input.** A `CanonicalModel` from `@iap/model`'s `canonicalize` or any `{resources, policies}` pair in that shape — policies evaluate against the canonical, defaults-applied document only (ch. 7 §7.6): a Database that omits `encryption` still evaluates with `spec.encryption.atRest: required` (tested).
- **Scoping/targeting** (ch. 7 §7.2): a resource is targeted when it matches both `target.kinds` (if present) AND `target.selector` (kinds + exact-match labels, if present); `target: {}` targets every resource. Roadmap scopes map onto this document-relative machinery: kind/resource scopes are `kinds`/labels, environment/profile scope is the profile-relative canonical model, org/BU/team/project scopes are pack concatenation (distribution itself is Phase 16).
- **Condition trees** (ch. 7 §7.3): `allOf`/`anyOf`/`not` to any depth over `{field, operator, value}` leaves; `field` is a dot path from the resource entry root (`kind`, `labels.*`, `spec.*`, `x-*` annotations). Unresolved paths: `absent` → true, `exists` → false, all other operators → false. No short-circuiting inside combinators, so IAP504 diagnostics are complete and order-stable.
- **All nine operators** (ch. 7 §7.4): `equals` (deep JSON equality), `not-equals`, `in`, `not-in`, `exists`, `absent`, `greater-than`, `less-than`, `matches`. Ordered comparisons use exactly three domains — numbers, quantities, durations — via `@iap/model`'s exact-rational BigInt parsers (`parseQuantity` milli-units, `parseDuration` milliseconds; no floating point anywhere). Operands in different domains produce a **IAP504** warning and the leaf evaluates **false**. A string that parses in both grammars (only possible with the shared `m` suffix, where both readings are strictly monotonic in the mantissa, so the order is identical either way) is compared as a quantity — documented in code.
- **Effects** (ch. 7 §7.5): `deny` → **IAP501** error where the condition holds; `warn` → **IAP503** warning where it holds; `require` → **IAP502** error where it does _not_ hold. `require` rules that are an `equals` leaf or an `allOf` conjunction of `equals` leaves (nested `allOf` allowed — still a pure conjunction) emit a deterministic **autofix**: an RFC 7386 merge patch setting each field to its value, returned in `PolicyResult.autofixes` and never applied silently. All other shapes are report-only.
- **Determinism** (ch. 7 §7.6): policies in lexicographic `id` order, resources in lexicographic resource-ID order (code-point comparison shared with canonicalization); findings/evaluations/autofixes are byte-identical across runs (tested via `JSON.stringify` equality). The result also carries a full **evaluation trace** — one `{policyId, resourceId, matched, verdict}` entry per (policy, resource) pair with verdicts `not-targeted`/`pass`/`violation`/`exempted`.

### M9.2 — Pack format + registry (`POLICY_PACKS`)

The pack format IS the ch. 7 policy format: a pack is a named array of schema-valid `Policy` objects, activated by concatenation into the evaluated set — no second engine, packs stay auditable as plain data. `POLICY_PACKS` (`src/packs.ts`) is the in-process registry with six built-in packs, every policy id prefixed by its pack name:

| Pack                  | Rules                                                                                                                                                |
| --------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `production-baseline` | require `spec.availability` = `high` on Database; require `spec.resilience.backup` = `required` on Database                                          |
| `encryption-baseline` | require `spec.encryption.atRest` AND `.inTransit` = `required` on Cache/Database/ObjectStore/Queue/Topic/Volume (allOf of equals — autofix-eligible) |
| `private-only`        | deny `spec.exposure` = `public` on all kinds (catches the Gateway default)                                                                           |
| `backup-baseline`     | require `spec.resilience.backup` = `required` on Database/ObjectStore/Volume                                                                         |
| `tagging-baseline`    | require `labels.costCenter` exists on all resources (resource-level path)                                                                            |
| `logging-baseline`    | require `spec.observability.logs` = `required` on Service/Database/Gateway/Function/Job                                                              |

Each pack targets only kinds that actually carry the referenced field in the normative schema, so packs never produce vacuous unresolved-path findings. A test validates every pack policy against `$defs/common/policy` of the embedded normative schema (ajv 2020-12, strict, `x-iap-*` vocabulary) and evaluates every pack against all 9 official examples. Roadmap packs needing later-phase inputs are deferred with their phases: approved-geography (region labels are a documented ch. 7 worked example, org-specific values), budget-limits (IAP505, Phase 10 cost annotations), development baseline and HA baseline variants (profile guidance, Phase 16 distribution).

### M9.3 — Exception workflow model

`PolicyException = {policyId, selector?, reason, approver, expiry, ticket?}` — scope, reason, approver, expiry, and ticket/evidence per the roadmap requirements; the audit trail is the finding message itself:

- An **unexpired** exception matching a violation (by policyId, and selector kinds/labels when present) downgrades the error finding to a **warning** with `[exception: <reason> approved by <approver> until <expiry>]` appended; the trace verdict becomes `exempted`. Exceptions remain visible — never silent suppression.
- An **expired** exception is ignored and reported as an **IAP503** warning (`expired exception: …`) so stale grants surface until removed. Unparseable expiry can never be shown active → treated as expired.
- Exceptions apply to error findings (deny/require); `warn` findings are already non-blocking.

## Architecture decisions

1. **`now` is injected, never read.** `Date.now()` in library code would make evaluation time-dependent, violating ch. 7 §7.1 determinism ("a policy file audited today evaluates identically forever"). `evaluatePolicies` takes `options.now` (ISO 8601) and **throws `TypeError` when exceptions are supplied without it** — there is no default: silently treating exceptions as unexpired (or expired) would change governance outcomes based on an implicit assumption. Hosts (CLI/control plane) own the clock.
2. **`matches` is a documented RE2 subset on the JS RegExp engine.** Ch. 7 mandates RE2 semantics; shipping a real RE2 binding would add a native/WASM dependency to an otherwise dependency-free engine. Instead, patterns using constructs RE2 rejects by design — backreferences `\1`–`\9` and lookbehind `(?<=`/`(?<!` (escape-aware scan) — are refused with an IAP504-style warning and the leaf evaluates false; invalid patterns likewise degrade to IAP504 + false, never a throw. Limitation: other non-RE2 constructs (e.g. lookahead) are currently accepted, and JS matching is not linear-time. A linear-time RE2 engine can replace this without changing behavior for RE2-valid patterns.
3. **IAP504 reused for unsupported/invalid patterns.** The registry defines IAP504 for ordered-comparison operand mismatches; an unsupported `matches` pattern is the same failure class (leaf cannot be evaluated soundly → warning + false). Reported as IAP504 rather than inventing an unregistered code; a future IEP may split out a dedicated code.
4. **Dual-grammar strings (`500m`) compare as quantities.** The only suffix shared by the quantity and duration grammars is `m`; when both operands parse in both grammars, both readings are strictly monotonic in the mantissa, so the comparison result is identical — quantity is chosen deterministically and documented.
5. **Exceptions live outside the document.** They are organizational state (approvals, tickets), not intent; keeping them caller-supplied preserves the canonical document/hash and matches the Phase 16 control-plane distribution model.

## SDK integration

`@iap/sdk` gained a `policies(options?)` method on `IaPWorkspaceResult` — evaluates the document's own `policies` array against the canonical model (memoized for the no-options call; option-carrying calls evaluate fresh) — and re-exports `evaluatePolicies`/`POLICY_PACKS` plus the public types. The frozen API-surface test was extended deliberately (§21.6: minors may add). Because `policies()` consumes only the canonical model, policy evaluation is pre-mapping and pre-planning by construction.
