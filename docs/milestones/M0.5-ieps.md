# Milestone M0.5 — Initial IEPs 0008–0013

**Phase:** 0 — Foundation and Repository Baseline (feeds Phase 0.5 exit criteria)
**Milestone:** M0.5
**Status:** Completed
**Date:** 2026-07-10

## Implemented

Drafted the six forward-looking IaP Enhancement Proposals required by roadmap §17.7, each following the full 27-section IEP template (see `spec/ieps/IEP-TEMPLATE.md`), Status: Draft:

- **IEP-0008 — Canonical Infrastructure Model.** The normalized in-memory representation between IaP documents and all engines: TypeScript logical schema, seven invariants (loss-free round-trip, canonical-form hash identity per spec ch. 1 C1–C6, provenance totality, extension preservation, immutability). Also owns the two determinism gaps from the gap report (quantity normalization, default materialization).
- **IEP-0009 — Intent compiler operations.** The 12 typed operations and operation envelope enforcing "LLMs never write YAML"; operations are validated then applied to the CIM.
- **IEP-0010 — State and reconciliation.** Pluggable state backend interface (local encrypted + remote with lease locks, fail-closed), roadmap §4.4 history record, and a unified two-axis drift taxonomy reconciling spec ch. 14 with roadmap Phase 14.
- **IEP-0011 — Deterministic planning contract.** Closed nine-input determinism vector, `plan.iap.dev/v1` artifact sketch (content-hashed planId, signed envelope with expiry outside plan content), risk scoring, invalidation, golden-plan CI.
- **IEP-0012 — Provider conformance.** Signed plugin manifests, load-time fail-closed coverage checks, declarative capability assertions, double-run hash equality, core/execution/drift certification levels, mock provider as reference harness.
- **IEP-0013 — AI and MCP trust boundaries.** Structurally disjoint authoring vs deployment tool registries, knowledge-snapshot citation/staleness model, acceptance → compiler-operation rule, injection defenses, seven trust-boundary conformance checks.
