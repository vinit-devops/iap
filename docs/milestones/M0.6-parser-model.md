# Milestone M0.6 — Minimum Parser and Model Packages + Automated Spec Validation

**Phase:** 0 — Foundation and Repository Baseline (roadmap §17.8–§17.9)
**Milestone:** M0.6
**Status:** Completed
**Date:** 2026-07-10

## Implemented

- **`@iap/model` 0.1.0** (`packages/model/`) — Canonical Infrastructure Model types for `iap.dev/v1`: `IaPDocument`, `ResourceEntry`, `RelationshipEdge`/`RuleEdge`, `Policy`/`PolicyCondition`, `Profile`, `Output`, `Finding`, plus normative constants (`KINDS`, `RELATIONSHIP_TYPES`, `POLICY_OPERATORS`, `COMPLIANCE_FRAMEWORKS`, `RESOURCE_ID_PATTERN`) and embedded copies of both normative schemas with accessor functions. Per ADR-0002, the package maintains **no second type system that can drift**: `tools/schema-generation/sync-schemas.mjs` copies the spec schemas at build, and tests assert (a) byte-equality of embedded vs `spec/schema/` copies and (b) every exported constant equals the corresponding schema enum.
- **`@iap/parser` 0.1.0** (`packages/parser/`) — `parseText` (YAML 1.2/JSON, duplicate-key rejection, single-document enforcement, non-object-root rejection, parse errors with `file:line:col` positions), `validateSchema` (Phase 1 schema validation producing `IAP101/102/103` findings), `loadDocument` (parse + apiVersion gate + schema validation), and `createValidator` — ajv 2020-12 in **strict mode** with the `x-iap-*` annotation vocabulary registered per spec ch. 24 CV-6 (no global strict-mode opt-out; `allowUnionTypes` enabled for the schema's union-typed policy params).
- **Spec-validation harness** (`tests/conformance/run.mjs`, `pnpm run test:spec`) — compiles both schemas, validates all 4 official examples + the reference mapping, asserts every conformance case produces its declared `# expected:` outcome, and shape-checks `ROADMAP.yaml`. Wired into CI (`.github/workflows/ci.yml`).

## Architecture decisions

- Schemas are **embedded by build-time copy + byte-equality drift test** rather than runtime reads outside the package boundary — keeps the published package self-contained while spec/schema stays the single source of truth (ADR-0002).
- Ajv strict mode stays ON with an explicitly registered annotation vocabulary — implementations get the spec-mandated behavior without weakening schema checking.

## Specification gaps

None new. The harness's "deferred to semantic validator" lines make the gap report's semantic-coverage point visible on every run.

## Security findings

None. No new runtime dependencies beyond `ajv` and `yaml`; no install scripts.

## Known limitations

- Parser emits source positions for parse errors only; per-node source maps (needed by the LSP) come with Phase 2's full parser milestone (M2.1).
- Validation covers Phase 1 (schema) only; reference/relationship/dependency phases (IAP2xx–IAP4xx) are Phase 2 engines (M2.4/M2.5).
- Canonicalization and hashing intentionally absent — blocked on IEP-0008's resolution of the two determinism gaps.
