# Milestone M1.5 — Error-Code Registry and Schema Compatibility Baseline

**Phase:** 1 — IaP v1 Stabilization
**Milestone:** M1.5
**Status:** Completed
**Date:** 2026-07-10

## Implemented

- **`spec/conformance/error-codes.yaml`** — machine-readable registry of all 32 IaP error codes (code, phase, severity `error|warning|contextual`, stage `validation|plan-time`, title). Chapter 8 remains the prose authority; the registry is the tooling-consumable projection.
- **Harness cross-check** — `pnpm run test:spec` now verifies, on every run, (a) bidirectional set equality between registry codes and every `IISnnn` mentioned in chapter 8, and (b) per-entry shape including that each code's leading digit equals its phase. This found and fixed one real gap: IAP805 (deprecated element) was defined in chapter 10 but absent from chapter 8's phase-8 list — added, keeping chapter 8 the single authority.
- **`docs/reports/schema-compatibility-1.0.md`** — the 1.0.0 compatibility baseline: frozen surfaces (kind registry, verb set, top-level keys, operators, grammars, annotation vocabulary, error codes), the M1.2 revision audit (annotation-only, verified non-breaking), and the chapter-10 compatibility promises restated as checkable statements.

## Architecture decisions

Registry lives under `spec/conformance/` (normative-adjacent, versioned with the spec) rather than `docs/` — implementations consume it (CLI `--output sarif` code mapping, LSP diagnostics, Phase 2 validators).
