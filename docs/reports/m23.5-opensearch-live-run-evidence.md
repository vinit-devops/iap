## Honest scope notes

- **Cheapest config, single-AZ.** One `t3.small.search` node, no dedicated masters, no zone
  awareness, gp3 10 GiB — the minimum billable posture. No fine-grained access control; the default
  restrictive access policy denies anonymous `es:*` (a real plan injects an account/role-scoped
  policy).
- **AWS-owned encryption key.** Encryption at rest is on with the AWS-owned `aws/opensearch` key
  (no customer-managed KMS pinned); the desired-gated compare means that default key is never
  misread as drift.
- **SearchIndex kind, no mapping emission yet.** The IaP SearchIndex kind does not emit an
  `aws:opensearch:Domain` mapping yet (per-branch outputs grammar gap, deferred like the M22.x
  handlers); this run drove the `AwsExecutor` directly with a hand-built `ProviderPlan`. No index
  mapping / template is emitted by IaP — the domain is provisioned, not its search schema.
- **engineVersion immutable in v1.** A version bump is really `UpgradeDomain` / blue-green, not an
  `UpdateDomainConfig` field; v1 treats it as immutable → gated replace (ADR-0006). Wiring the
  managed upgrade path is a later refinement.
- **Slow create/delete.** ~10-15 min each way (~30 min round-trip); no in-handler waiter — the live
  driver polls `DescribeDomain` out of band with bounded budgets.
- **Tests 3 & 5 execution justified-deferred.** Both need a multi-minute blue/green
  `UpdateDomainConfig` that would extend the billable lifetime; classification (test 3) and the
  read-only drift mechanism (test 5) are proven live, execution is covered by the green mock suite.
- **Teardown orchestration.** Domain teardown (tests 6-9) completed by the coordinator via the
  handler driver (`node driver.mjs destroy`) after the agent's create-poll stalled on a session
  token blip; the domain reached ACTIVE with a live endpoint (test 1) before teardown; zero orphans
  verified. The captured executor output (genuine `AwsExecutor` JSON — the verbatim managed-only
  refusal and the applied managed delete) is quoted in the test table above, and this run's own
  delete-poll independently observed `ResourceNotFoundException`.
- **Parallel-wave sweep.** Two sibling M23.5 per-service runs (Kinesis, Firehose) were live in the
  same account/region, so the region-global `live:sweep` was NOT run here — cleanup verification is
  run-scoped by the plan-id substring `1784351931`; the coordinator runs the global sweep after the
  wave.
