## Real bugs caught by live execution (mock tests could not)

1. **Scheduler tags apply to schedule GROUPS, never schedules** — TagResource/ListTags reject
   schedule ARNs (`…:schedule/default/…` fails the `schedule-group/` ARN pattern). The managed
   ownership contract therefore rides a **handler-owned, tagged schedule group** named after
   the resource; the schedule lives inside it and is deleted with it. Cron coercion
   (5-field → AWS 6-field with dom/dow exclusivity) held live on first contact.
