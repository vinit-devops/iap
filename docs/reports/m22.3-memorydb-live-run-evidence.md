## Honest scope notes

- **Default VPC (ADR-0005):** the handler placed the cluster in the account default VPC
  (`vpc-033b2dc4f769d7f77`) via its own subnet group `infraasprompt-1784338124-mem-subnets` over 3 AZ subnets —
  the M21.3 ElastiCache live-fix pattern. M23.4 Network handlers replace this pragmatism.
- **No mapping emission** for the Cache kind — the run drove the executor directly with a hand-built
  ProviderPlan (same gap as M22.1 apigatewayv2 / M22.2 DynamoDB / M22.3 DocDB & Neptune).
- **Drift via `aclName` (pending-update honesty):** MemoryDB has no *fast* out-of-band projection
  mutation — `nodeType`/`replicas` are slow (tens of minutes), `tlsEnabled` is immutable (replace),
  and `engineVersion` is intentionally desired-gated. `aclName` was the honest choice, but it flows
  through MemoryDB's pending-update model: the raw-CLI change is not reflected in the readable projection
  until it converges, so the drift + read-only-plan + reconcile cycle each required a convergence wait
  rather than a single instant round-trip. The core safety property — the plan/drift surface is strictly
  read-only — was proven (live `ACLName` unchanged after the plan).
- **Drift-helper ACL straggler:** the drift test created a raw-CLI helper ACL `infraasprompt-1784338124-acl`
  (empty, **untagged**, created out-of-band, not IaP-managed). It was disassociated from the cluster by
  the reconcile. Because it carries no `iap:*` tags, the run-scoped *tag* sweep (Test 9 → zero) does not
  cover it; it must be removed by the wave's name-prefix sweep (`infraasprompt-1784338124-*`, which also catches
  IAM/ACLs the tag API misses). Flagged honestly: this agent made no AWS calls after teardown and could
  not delete it directly.
- **Replacement (Test 4)** executed as refusal-only, justified above.
- **Cut-order-lean / fresh re-run:** single cluster, 1 shard, `db.t4g.small`; scaled to 1 replica for
  Test 3 and left there for teardown. This run is a **clean re-run after two prior token-expiry
  interruptions** whose resources were torn down first — it started from an empty account.
- **Parallel wave:** 5 sibling M22.3 service runs were live in the same account/region concurrently, so
  the region-global `live:sweep` was NOT run here — cleanup verification was **run-scoped by run id**
  (`1784338124`); the coordinator runs the global sweep after the wave.
- **Coordinator finalization:** Tests 6–9 were completed by the coordinator via the handler driver after
  this agent's convergence poller stalled; teardown was verified zero-orphan (run-scoped). Tests 1–5
  evidence is from this agent's own live execution.
