## Honest scope notes

- **`.internal` public test zone:** a public hosted zone named `…iap-live.internal` — deliberately
  non-resolvable, chosen purely for Route 53 CRUD lifecycle testing. It proves nothing about DNS
  resolution, only about zone/record management, which is the point.
- **Replacement executed as refusal-only (justified):** flipping `public`→`private` is a genuine
  replace, but a private zone needs a VPC association the handler does not emit, so executing the
  destructive delete+create would strand the resource. The gate-closed refusal is the load-bearing
  contract and it was proven verbatim; the destructive execution mandate was already met live in
  M22.2.
- **No mapping emission** for the DnsZone kind — the run drove the executor directly with a
  hand-built ProviderPlan (same per-branch-outputs grammar gap deferred since M22.1).
- **Drift key:** for a HostedZone only `comment` is a mutable projected attribute; `zoneName`/
  `visibility` are immutable (→ replace) and tags/records are outside the drift projection, so drift
  detect+reconcile was exercised via an out-of-band `comment` change — the honest, projection-true
  path.
- **Parallel wave:** 5 sibling M23.2 service runs were live in the same account/region concurrently,
  so the region-global `live:sweep` was NOT run here — cleanup verification was **run-scoped by run
  id** (`1784345750`); the coordinator runs the global sweep after the wave.
