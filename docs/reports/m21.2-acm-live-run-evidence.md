## Real bugs caught by live execution (mock tests could not)

1. **`example.com` certificates fail instantly** — the first attempt used
   `<runid>.example.com`; the reserved domain's CAA policy drove the request to **`FAILED`**
   (not the expected `PENDING_VALIDATION`). Lesson recorded: live-run cert domains must be
   syntactically valid but **unregistered** domains. The FAILED cert was destroyed cleanly.
2. **A `domainName` change plans `create`, not `replace`** — the handler locates certificates
   *by domain name*, so changing the domain changes the lookup key: the read finds nothing,
   plans `create`, and would orphan the old certificate. The reachable replace surface is
   `validationMethod` drift (same domain). Known limitation carried to **M23.2** (ACM as
   first-class Certificate), where identity should pin to the stored ARN from state instead of
   a domain lookup.
3. **ACM list consistency lag** — `list-certificates` can briefly show a just-deleted
   certificate; cleanup verification must re-check (or query with explicit statuses).
