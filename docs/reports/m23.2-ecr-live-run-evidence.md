(`--confirm` equivalent = `apply: true`), all resources tagged `iap:managed=true` / `iap:planId` /

PutImageTagMutability / PutImageScanningConfiguration for in-place updates, TagResource to
re-assert ownership, DeleteRepository `force:true` for teardown. No mapping emission for the

## Honest scope notes

- **No mapping emission** for the container-Registry kind — the run drove the executor directly
  with a hand-built ProviderPlan (same gap as M22.1/M22.2/M22.3).
- **Cut-order-lean:** a single empty repository, no images pushed. `force:true` teardown was
  therefore untested against actual image content (the un-forced-delete-fails path is covered by
  mock only); the flag is present and the delete succeeded.
- **KMS key:** the replacement used the AWS-managed default ECR KMS key (no customer key pinned).
  The handler's desired-gated kmsKey comparison means an unpinned plan does not read the
  AWS-assigned key as drift (M22.1/M22.2 lesson) — confirmed: post-replacement no-op held.
- **Parallel wave:** 5 sibling M23.2 service runs were live in the same account/region
  concurrently, so the region-global `live:sweep` was NOT run here — cleanup verification was
  **run-scoped by run id** (`1784345751`); the coordinator runs the global sweep after the wave.
