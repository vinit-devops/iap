## Real bugs caught by live execution (mock tests could not)

1. **ACM rejects reserved/invalid domains → `FAILED`, never any validation record; the handler
   burned its whole poll budget on a misleading timeout.** The first (`.internal`) create
   requested the cert, then `waitForValidationRecords` polled 60× (~15 s) and threw *"did not
   expose DNS validation records within 60 polls"* — masking the real cause. Direct
   `describe-certificate` showed `Status: FAILED`, `FailureReason: INVALID_PUBLIC_DOMAIN`, empty
   `ResourceRecord`. Mocks always return records, so only live ACM reaches this path.
   **Fix:** `waitForValidationRecords` now fail-fasts on `Status === 'FAILED'`, surfacing the
   `FailureReason` instead of a timeout (`packages/deploy-aws/src/acm.ts`). **Regression test:**
   `packages/deploy-aws/test/m23.2-acm.test.ts` ("create fails FAST when ACM rejects the domain").
   Full `@iap/deploy-aws` suite green after the fix: **364 tests, 38 files.**
