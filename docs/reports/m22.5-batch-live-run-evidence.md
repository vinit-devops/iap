## Real bugs caught by live execution (mock tests could not)

1. **Disable-then-delete self-race (JQ + CE).** First destroy pass, verbatim on both:
   `Cannot delete, resource is being modified.` Cause: the handlers issued
   UpdateJobQueue/UpdateComputeEnvironment `State=DISABLED` and the Delete immediately after;
   live Batch holds the resource in status UPDATING while the disable lands and rejects a
   delete mid-modification â€” so a single destroy pass could **never** delete an ENABLED JQ/CE.

   disable-**settle**-delete â€” after the disable, poll Describe\* until state DISABLED and

   delete), then delete. Regression tests:
