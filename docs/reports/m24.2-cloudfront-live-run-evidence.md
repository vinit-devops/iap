## Disable-then-delete teardown detail (the roadmap's longest wall clock)

1. Distribution created `Enabled=true`, reached `Deployed` (~2.5 min).
2. `destroy` #1 → `UpdateDistribution(Enabled=false, IfMatch=<fresh ETag>)` accepted → `Status=InProgress`.
   The same handler call's `DeleteDistribution` was rejected ("has not been disabled" — the disable
   had not yet propagated). This is the documented behavior the live driver retries.
3. Polled `GetDistribution` until the disable propagated to `Status=Deployed`.
4. `destroy` #2 → handler saw `Enabled=false`, skipped the redundant disable, issued
   `DeleteDistribution(IfMatch=<fresh post-disable ETag>)` → accepted.
5. Polled `GetDistribution` → `NoSuchDistribution` — distribution confirmed **gone**.

Every mutation used an ETag fetched FRESH via `GetDistribution` immediately before the call (never
cached from read) as the `IfMatch` optimistic-concurrency token, mirroring wafv2's LockToken.
