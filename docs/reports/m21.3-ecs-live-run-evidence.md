## Real bugs caught by live execution (mock tests could not)

1. **ECS service-linked role first-use race** — the account had never run ECS in this region;
   `CreateService` failed with **`Unable to assume the service linked role`** before
   `CreateCluster`'s implicit SLR creation propagated. Fixed operationally
   (`iam create-service-linked-role --aws-service-name ecs.amazonaws.com`); handler-level
   ensure-SLR is a candidate improvement for M22.5.
2. **`Assign public IP is not supported for this launch type`** — the first gated replacement
   (FARGATE→EC2 keeping `assignPublicIp: true`) failed AFTER the delete phase. The executor
   recorded the honest half-state (`errors[]`, exit 3, no silent success) exactly as ADR-0006
   specifies, and a plain re-apply recovered. Lesson: EC2 launch type rejects
   ENABLED public-IP assignment in awsvpc config.
3. **Tagging-index ghosts** — the Resource Groups Tagging API retains DELETED (INACTIVE) ECS
   services/clusters; the sweep initially flagged 2 false orphans. The sweep now verifies
   `ecs:` ARNs against actual ECS status (`tools/live-run/sweep.mjs`).
4. **DRAINING → cluster-delete race (pre-empted)** — a bounded INACTIVE waiter was added to
   the handler before the run; it held live (cluster deleted cleanly after force-delete).
