## Real bugs caught by live execution (mock tests could not)

1. **`Redis Auth can only be enabled within an Amazon VPC`** — the handler created the RG
   without a cache subnet group. Fix: the handler now owns a subnet group
   (`<id>-subnets`) over the default VPC — created before the RG, deleted after the RG is
   gone (bounded 10-min waiter). Regression-tested in `m21.3-handlers.test.ts`.
2. **`Replication group must have at least one read replica to enable autofailover`** —
   `update()` modified failover before increasing replicas. Fix: replica count reconciles
   FIRST, and failover waits for the next convergence pass when replicas changed (replica
   creation is async).
3. **`Cluster not in available state to perform tagging operations`** — during modify windows
   the tag API rejects reads, which crashed a READ-ONLY drift check. Fix: `read()` tolerates
   `InvalidReplicationGroupState` on the tag call; unknown tags read as unmanaged, so destroy
   stays fail-closed (the safe bias).
